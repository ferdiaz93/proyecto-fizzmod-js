Prueba Front End

A tener en cuenta:

  • La prueba se puede hacer en JS nativo, o también se puede utilizar React.
  • Para escribir los estilos se puede usar CSS o SASS/SCSS/LESS.
  • En caso de usar React está permitido el uso de las herramientas para hacer un quickstart o generar el boilerplate inicial como create-react-app.
  • Escribir un solo archivo para toda la aplicación o separarlo por módulo o componente.
  • Preferentemente escribir el código usando la version ECMAScript 6 (ES6) o superior con sus features (const, let, arrow functions, Class, Promise, los métodos de Array.prototype, también se puede utilizar async/await de ES7, etc).
  • Se puede usar un bundler/compilador/task runner para transpilar el código a ES5 como Webpack, Rollup, Gulp, Grunt, etc.
  • Se pueden utilizar paquetes de NPM para simplificar las tareas.

Credenciales para acceder a invision

  • Crearse una cuenta de invision a traves del link que le fue enviado via mail para poder acceder a los diseños de la prueba.
  • Utilizar la herramienta inspect de invision para consultar los estilos del diseño.

Invision es la plataforma donde subimos los diseños de nuestros proyectos, los cuales podes inspeccionar.

Enunciado

La maqueta consiste en 3 secciones principales Header, Body y Footer.

El header consiste en las siguientes secciones:

  • Logo
  • Topnav
  • Buscador
  • Carrito
  • Menú/Navegador

El body consiste en las siguientes secciones:

  • Banner Slider
  • Listado de producto con titulo y filtros

El footer consiste solo en un bloque referencial:

  • Bloque footer (solo texto y fondo)

En donde se vea un ícono de la manito se indica que el elemento tiene “hover” por lo tanto hay que estilizarlo como indica el diseño.

Header

Las categorías del menú serán solicitadas via Ajax al siguiente endpoint: http://remote.fizzmod.com/menu.json

Menú de categorías

  • Recorrer las categorías dentro de menu.categories en el JSON recibido y generar un link de cada categoría usando el title y el href.

Resto del header

Recomendaciones:

Logo

Utilizar el asset logo_fizzmod.svg.

Topnav

Los elementos del mismo deben ser links.

Body

El contenido del body será solicitado via Ajax al siguiente endpoint: http://remote.fizzmod.com/body.json

Banner Slider

  • Para generar el banner recorrer slides del JSON recibido y generar un <a> con el link indicado y dentro de la misma crear una <img> con la imagen indicada.

Listado de productos

  • Para generar los productos recorrer products del JSON y generar un producto por cada object.
  • Por cada producto verificar si hay descuento segun el precio y generar de forma dinámica el globo con el porcentaje del descuento. Si no hay descuento no mostrar esta burbuja.
  • El precio tachado de los productos se debe aplicar solo a aquellos que tengan diferecia de precios.
  • Todo el elemento debe ser clickeable y debe linkear a la url referenciada en el href del object de producto.

Filtros de productos

  • Para armar los filtros se debe utilizar la propiedad del JSON filters que ya tiene los filtros precalculados y guardados por grupos.
  • El título de cada agrupación de filtros debe ser su correspondiente title.
  • Los filtros tienen que ser “checkbox” y tienen que verse como impone el diseño utilizando los assets.

OPCIONAL

  • Implementar la funcionalidad para que filtre la visibilidad de los productos según los filtros asignados. En el caso de que no haya resultados posibles, dejar en blanco.
